# OWASP Game Security Framework (OGSF)

[![OWASP Incubator](https://img.shields.io/badge/owasp-incubator%20project-48A646.svg)](https://owasp.org/projects/)
[![Creative Commons License](https://img.shields.io/github/license/OWASP/www-project-gamesec-framework.svg)](https://creativecommons.org/licenses/by-sa/4.0/ "CC BY-SA 4.0")

Games today are complex, connected ecosystems. They combine game clients, backend services, player identities, social features, virtual economies, live operations, and in some cases highly competitive or esports-like environments.

The **OWASP Game Security Framework (OGSF)** is a community-driven project focused on helping the industry define, review, and improve security requirements for modern games and connected game systems.

OGSF is intended to support game developers, security practitioners, testers, architects, and technical leaders by providing a practical foundation for understanding common game security risks and improving how they are addressed. The framework aims to help teams reduce vulnerabilities, improve trust boundaries, strengthen game integrity, and support fair and secure player experiences.

Rather than treating games as only traditional applications, OGSF looks at the broader game ecosystem, including areas such as multiplayer interactions, backend trust, client integrity, connected services, abuse scenarios, and virtual economy risks.

## Deliverables

### Core Framework
The main OGSF framework provides structured security requirements and guidance for modern games and connected game ecosystems. It is intended to serve as the central project deliverable.

### Verification Guidance
Practical guidance to help teams review, interpret, and apply framework requirements in real development and testing scenarios.

### Threat and Scenario Coverage
Focused material covering common game security concerns such as multiplayer abuse, client trust boundaries, backend and API risk, cheating-related security issues, and virtual economy concerns.

### Worked Examples
Illustrative examples showing how framework concepts can be applied in practice to game systems and game-connected services.

### Mappings and Related References
Where useful, the project may include mappings or relationships to relevant OWASP materials and other supporting security resources.

## Project Goals

The project aims to:

- provide practical and understandable security guidance for the game industry
- improve consistency in how game security requirements are defined and reviewed
- support secure design, development, testing, and release practices
- help teams think beyond isolated vulnerabilities and consider the full game ecosystem
- encourage community review and shared learning across game development and security disciplines
